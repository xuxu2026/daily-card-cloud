# 日签卡片测试方案

## 一、测试目标
验证日签卡片的各项功能是否正常，确保推送的内容准确、美观。

---

## 二、测试维度

### 1. 数据准确性测试

| 测试项 | 验证内容 | 预期结果 |
|--------|----------|----------|
| **天气数据** | 温度、天气状况、风力 | 与实际天气一致 |
| **生活指数** | 运动、洗车、太阳镜、过敏、舒适度 | 有数据且合理 |
| **空气质量** | AQI数值、等级描述 | 格式正确（空气 XX 优/良/轻度污染） |
| **限号信息** | 今日/明日限号尾号 | 符合成都限行规则 |
| **美文内容** | 两条美文是否不同 | poem1 ≠ poem2 |
| **护肤贴士** | 季节性贴士 | 与当前季节匹配 |

### 2. 显示逻辑测试

| 测试项 | 验证内容 | 预期结果 |
|--------|----------|----------|
| **美文显示** | 短美文(<22字)显示2条，长美文(≥22字)显示1条 | 符合预期 |
| **图标显示** | 洗车=🚗<sup>🚿</sup>，其他生活指数图标正确 | 符合预期 |
| **生活指数居中** | 5个指数标签居中显示 | 居中对齐 |
| **穿衣提示** | 格式为"穿衣 XX" | 正确简化 |
| **紫外线提示** | 格式为"紫外线 XX" | 正确简化 |
| **空气指数** | 格式为"空气 XX 等级" | 完整文字描述 |

### 3. 主题样式测试

| 测试项 | 验证内容 | 预期结果 |
|--------|----------|----------|
| **72种常规主题** | 按一年中第几天轮换 | 两个多月不重复 |
| **32种节日主题** | 节日期间优先使用 | 配色/暗纹与节日关联 |
| **字体加载** | Google Fonts正常加载 | 无字体加载错误 |
| **暗纹装饰** | 30种通用+18种节日专属 | 图案清晰可见 |
| **LOGO显示** | 夏天抗衰诊所LOGO | 左上角、40×120px、透明度0.85 |

### 4. 推送功能测试

| 测试项 | 验证内容 | 预期结果 |
|--------|----------|----------|
| **企业微信推送** | Webhook发送成功 | 群机器人收到图片 |
| **图片格式** | PNG格式、750×1100px | 分辨率正确 |
| **文件保存** | output/daily_card.png | 文件存在 |

## 三、测试用例执行

### 场景1：正常工作日测试
```bash
python main.py
```
- 检查天气、指数、美文是否正常显示
- 验证推送是否成功

### 场景2：美文逻辑测试
```bash
# 修改 poem1 为短句，验证显示2条
# 修改 poem1 为长句，验证显示1条
```

### 场景3：连续多天测试
```bash
# 连续运行5天，观察：
# - 72种主题是否正常轮换
# - 美文是否每天不同
# - 生活指数是否有变化
```

### 场景4：边界条件测试
- 网络不稳定时的数据获取
- API返回异常数据时的处理
- 空值/None值的显示

### 场景5：节日主题测试
- 模拟不同节日日期，检查主题切换

## 四、测试检查清单

### 生成前检查
- [ ] 天气API正常连接
- [ ] 和风指数API正常连接
- [ ] 美文数据获取成功
- [ ] 限号规则正确

### 生成后检查
- [ ] 卡片标题显示正确日期
- [ ] 星期几显示正确
- [ ] 温度与实际天气一致
- [ ] 生活指数都有数据
- [ ] 美文不重复
- [ ] 洗车图标为🚗<sup>🚿</sup>
- [ ] 紫外线/穿衣/空气为完整文字
- [ ] 生活指数居中显示
- [ ] 主题配色正常
- [ ] LOGO清晰可见
- [ ] 无乱码/显示异常

### 推送后检查
- [ ] 企业微信收到消息
- [ ] 图片清晰无失真
- [ ] 排版正常
